这段时间有客户反应，mt8735_HDMI1080P 版本的开发板不支持USB camera，因此，做了简单的调试，使得mt8735_HDMI1080P 版本的开发板初步支持USB camera。

usb camera 就是底层支持 UVC，UVC全称是usb video class，一种usb视频规范。所有遵循uvc协议的摄像头都不需要安装额外的驱动，只需要一个通用驱动即可。Linux内核已经集成了uvc驱动，代码路径是kernel-3.10/drivers/media/usb/uvc/

调试过程：

一、打开配置

1.在~/mt8735/kernel-3.10/arch/arm64/configs/elink8735_tb_l1_defconfig文件下打开配置，支持UVC设备，支持OTG，这里使用了menuconfig进行了修改。

Linux内核需要打开以下配置来支持UVC设备：

CONFIG_MEDIA_SUPPORT=y

CONFIG_MEDIA_CAMERA_SUPPORT=y

CONFIG_VIDEO_DEV=y

CONFIG_VIDEO_V4L2=y

CONFIG_VIDEOBUF2_CORE=y

CONFIG_VIDEOBUF2_MEMOPS=y

CONFIG_VIDEOBUF2_VMALLOC=y

CONFIG_MEDIA_USB_SUPPORT=y

CONFIG_USB_VIDEO_CLASS=y复制代码MTK平台还需要额外打开OTG配置：

CONFIG_USB_MTK_OTG=y

CONFIG_USB_MTK_HDRC=y

CONFIG_USB_MTK_HDRC_HCD=y复制代码插入摄像头，打开USB调试模式，使用adb进入设备文件root@:/dev #，如果生成了/dev/video0设备节点，则证明UVC摄像头已经加载成功了。但是，我们的开发板上OTG接口和USB接口不能同时共用，那么接入USB Camera就不能使用USB调试，因此，需要使用网络adb。

二、网络adb的用法

1.打开USB调试模式，用USB连接adb服务

输入命令：adb tcpip 5555

手机就会显示：restarting in TCP mode port：5555

adb服务就会进入TCP模式，端口5555

四、USB Camera成像问题

1.安装一个测试软件

安装了一个测试软件SimpleWebCam.apk，但是并没有成像输出到屏幕上，可以使用网络adb查看日志信息。

SimpleWebCam.zip

点击文件名下载附件

2. 查看日志

(1)查看日志的方法：

查找关键字WabCam，输入命令：adb logcat | grep --color=auto  WebCam #设置匹配字符串颜色。

(2)查看日志

日志中有这么一项打印输出：

E/WebCam  ( 3352): Cannot open '/dev/video0': 13,Permission denied #缺少权限

因此，成功生成节点后，还需要为它添加权限：system/core/rootdir/ueventd.rc 下添加：

/dev/video0    0666   root    root

这是在mt8735系统中添加的，重新编译系统，将system.img下载到开发板上即可。

3.权限问题

本以为添加了/dev/video0    0666   root    root就可以了，但是，有时候video0会变成video1，第一次打开SimpleWebCam.apk软件的时候，图像正常，之后再次进入SimpleWebCam.apk，就没有图像了。查看了日志信息，显示没有找到设备。在查看root@:/dev #，dev/video0变成了dev/video1！USB摄像头并没有换，为什么会显示dev/video1。修改了SimpleWebCam.apk，使其支持video0~video3，但是日志输出E/WebCam  ( 3352): Cannot open '/dev/video1': 13,Permission denied，于是我们的解决方法如下：

system/core/rootdir/ueventd.rc 下添加：

/dev/video0    0666   root    root

/dev/video1    0666   root    root

/dev/video2    0666   root    root

/dev/video3    0666   root    root

解决了上述问题。
